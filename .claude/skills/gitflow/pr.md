# 절차 B — PR 생성
공통 전제·금지·중단 조건은 SKILL.md를 따른다. 기본적으로 직접 병합하지 않으며 병합은 사용자가 수행한다.

**위임 원칙** — 병합 및 병합 후 브랜치 정리는 원칙적으로 사용자가 수행한다. 단, 사용자가 해당 작업을 명시적으로 위임한 경우 대행할 수 있다. 대행 시에도 (a) 확인 게이트(7단계)는 그대로 적용하고, (b) 병합은 GitFlow의 squash merge 전략을 따르며, (c) 대행 여부·범위(병합만 / 브랜치 정리만 / 둘 다)를 최종 요약에 명시한다. 위임이 없거나 위임 범위가 불명확하면 대행하지 않고 사용자에게 넘긴다.

## 목표 상태
- feature/*·bugfix/*·chore/*: develop 대상 PR 생성됨, 작업 디렉터리 clean.
- hotfix/*: main·develop 각각 대상 PR 생성됨, 작업 디렉터리 clean.
- 브랜치 정리는 사용자가 PR 병합 완료 후 별도 수행(명시적 위임 시 대행 가능).

## 수행 절차
1. **현재 상태 확인** — 작업 디렉터리가 dirty면 즉시 중단하고 보고한다.
2. **원격 동기화** — 삭제된 원격 브랜치 참조를 포함해 모든 원격 상태를 최신화한다.
3. **베이스 최신화** — feature/*·bugfix/*·chore/*는 develop, hotfix/*는 main의 원격 최신 상태를 반영한다.
4. **작업 브랜치 리베이스** — feature/*·bugfix/*·chore/*는 develop 위로, hotfix/*는 main 위로 리베이스한다. 충돌 시 즉시 중단하고 충돌 파일 목록을 보고한다.
5. **변경 검토** — 베이스 대비 변경 파일 목록과 커밋 이력을 출력한다.
6. **테스트 실행** — 리포의 테스트 실행 방법을 스스로 파악해 실행한다(파악 불가 시 사용자 확인 후 진행).
   - CI/워크플로·빌드 설정 등 코드 실행 결과로 검증할 수 없는 변경(예: `.github/workflows/*`)은 "테스트"를 정적 검증 수준으로 정의한다: YAML 구문/스키마 유효성, 변경된 잡의 트리거·매트릭스·캐시 키 정합성 점검. 실제 동작은 PR 후 GitHub Actions로 검증되므로 로컬 검증 불가를 실패로 간주하지 않는다.
   - 실패 처리: 원인이 이번 작업 코드면 분석·수정 후 재실행(최대 2회). 2회 내 미해결이거나 원인이 CI 설정/환경/기존 코드면 요약 보고 후 즉시 중단.
7. **PR 생성 승인 요청 (확인 게이트)** — 다음을 제시하고 명시적으로 확인한다: PR 경로(feature/*·bugfix/*·chore/*→develop, hotfix/*→main 및 develop), 변경 파일·커밋 수, 테스트 결과. 병합·브랜치 정리 대행을 위임받은 경우 대행 범위도 함께 제시해 확인한다. 거부/무응답 시 즉시 중단.
8. **PR 생성** — 기본은 직접 병합 금지. PR 설명에 브랜치명·주요 변경 요약. feature/*·bugfix/*·chore/*: 작업 브랜치 → develop PR. hotfix/*: 작업 브랜치 → main PR 생성 후 → develop PR 생성. 병합은 사용자가 수행하되, **명시적으로 위임받은 경우 대행 가능**(squash merge 전략 적용, 7단계 확인 게이트 통과 필수).
9. **브랜치 정리** — 기본은 삭제하지 않는다(사용자가 PR 병합 완료 후 별도 수행). **단, 사용자가 명시적으로 위임한 경우** PR 병합 완료 후 작업 브랜치(및 해당 원격 브랜치)를 대행 삭제할 수 있다.

## 진행 보고
각 단계 완료 후: ✅ [N단계 완료] — [실행한 내용 한 줄 요약]
전체 완료 후 최종 요약:
```
## PR 생성 완료 요약
- 생성된 PR: [작업 브랜치] → [대상 브랜치]
- 변경 파일 수: N개 / 포함된 커밋: N개
- 테스트 결과: PASS / 수정 후 PASS(N회) / 정적 검증(CI·설정 전용 변경) / SKIP(파악 불가)
- 병합: 사용자 수행 예정 / 대행 완료(위임, squash)
- 브랜치 정리: PR 병합 후 사용자가 별도 수행 / 대행 완료(위임)
- PR URL: [URL] (hotfix/*는 main [URL] / develop [URL])
```
